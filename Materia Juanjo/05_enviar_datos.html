Nombre:

En una situación real el problema no es enviar un input malintencionado que descargue código malicioso con el resultado de infectarme a mi mismo.

El objetivo es camuflar llamadas javascript a otros servidores en el texto que se envía en un post, se almacena en una base de datos y se muestra, con el javascript disimulado, para que otros usuarios descarguen el código malicioso

Algunos ejemplos:

<big><big><big>Luisito</big></big></big> Modifica tamaño. Es una molestia

</div> Es una molestia pero si la página es compleja, puede cargarse la maquetación

Luisito<script>alert('Tonto, que eres tonto')</script> Hace que el cliente que entre en la página que lista algo así como los 10 últimos POSTs, ejecute un código javascript que supone un daño reputacional

Luisito<script src="https://ajax.googleapis.com/ajax/libs/d3js/7.8.5/d3.min.js"></script> Hace que otros clientes descarguen código potencialmente peligroso de otro servidor: Cross Site Scrtipting (XSS)
